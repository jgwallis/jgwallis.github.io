Linux First Actions

  1. Switch to a local mirror.
  2. Download and install all available updates.
  3. Go into the Driver Manager then download and install all available proprietary drivers.
  4. Install Microsoft’s core fonts.
  5. Install Google Chrome and possibly Chromium as well.
  6. Set all browser home pages as required.
  7. Set up e-mail accounts using Mozilla Thunderbird to avoid the constantly changing user interfaces, and advertisements, that are a common feature of webmail.
  8. Remove one-click application shortcuts from the left of the panel.
  9. Change the text scaling factor, perhaps to ‘1.5’, if required.
  10. Alter the desktop icon size if required, possibly to ‘large’.
  11. Put the trash bin icon on the desktop.
  12. Change the date and time format shown by the clock, if required.
  13. Set the desktop background to whatever the user wants.
  14. Install whatever other applications the user wants, such as LibreOffice, GIMP, Krita, Scribus, VLC Media Player, etc.
  15. If you’ve installed LibreOffice, set up desktop shortcuts to the most commonly used individual components, such as Writer, Calc, etc., then set the default fonts to ‘Times New Roman’ and ‘Arial’ if required.
  16. If dual-booting with Windows, set up automatic mounting of the Windows filesystem and create a symbolic link to the user’s Windows data.
  17. Turn off transparency for terminal windows.
  18. Consider reducing the system’s tendency to use swapping by editing ‘/etc/sysctl.conf’ and adding a line like ‘vm.swappiness=15’. Note that this change will not take effect until the system is rebooted.
  19. Turn on the firewall.
  20. Change the root password if necessary using ‘sudo passwd’.
  21. Forget any Wi-Fi networks that have been used to set up the computer, other than the customer’s own network.
